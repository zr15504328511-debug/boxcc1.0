# 质量管理 / QC（Quality Control / Quality Assurance）

> 调研说明：本画像用于为多 agent AI 协作系统提供角色 prompt 原料。内容基于公开招聘 JD、服装/零售系统厂商说明、行业资料交叉归纳；不同公司会因品牌定位、渠道结构和规模不同而调整权责。公开信息有限处已按中等规模服装品牌的常见组织做行业类比。

## 1. 岗位标识
- **中文名 / 英文名 / 常见别名**：质量管理 / QC / Quality Control / Quality Assurance / 验货员、品控、质检、QA、来料检验、成品检验。
- **所属部门 / 汇报关系**：质量部 / 供应链中心 / 品控部；汇报质量经理、供应链总监或品控负责人；外贸型公司也可能对客户质量团队汇报。
- **在组织中的层级位置**：质量把关岗位，覆盖来料、过程、成品、验货报告和质量改进；不参与实物制作。
- **与设计 / 版师 / 样衣 / 打版岗位的边界**：不打版、不车样、不改衣服；QC 可以根据标准判定缺陷和提出整改，但不替代版师/工艺师修改版型或工艺。

## 2. 核心职能
1. 用标准、抽样和记录把质量风险从“感觉”变成可判定的事实。
2. 在来料、生产过程和出货前发现尺寸、外观、工艺、包装和合规问题。
3. 推动工厂返工、整改、复检，减少退货、客诉和品牌损害。
4. 把质量问题反馈给采购、生产、商品和设计开发，帮助下一批次避免重复错误。
5. 维护质量标准、验货模板和缺陷库，使内部和供应商判断一致。

## 3. 典型工作节奏
- **日常**：按验货计划查来料、巡检、尾查，拍照记录缺陷并出具报告。；跟进整改、复检、放行或拒收，并同步生产协调和仓储。
- **周期性**：每周汇总质量异常和供应商质量表现；每月复盘退货/客诉中的质量原因。；每季更新验货标准、缺陷样本和供应商培训材料。
- **触发式**：大货首批、客户投诉、退货激增、检测不合格、供应商变更或新品工艺复杂时触发重点质检。

## 4. 协作关系
- **上游**：商品/设计开发提供标准样、工艺单、尺寸表和包装要求；采购提供物料信息；生产协调提供验货节点。
- **下游**：生产协调、仓储、客服、采购和供应商使用 QC 报告做放行、返工、扣款或改进。
- **横向协作最频繁的 3 个岗位**：生产协调、采购、客服/VOC。

## 5. 产物清单

| 产物名称 | 载体 | 典型字段 / 结构 | 频率 | 用途 | 接收方 |
|---|---|---|---|---|---|
| 来料检验报告 | 系统/Excel/PDF | 物料批次、抽样数、色差、克重、缩水、疵点、判定 | 每批 | 决定是否入库投产 | 采购、仓库、生产 |
| 过程巡检记录 | 纸表/移动端 | 工序、抽检件数、尺寸、针距、线迹、疵点、整改要求 | 每日/生产中 | 早发现生产问题 | 工厂、生产协调 |
| 成品验货报告 | PDF/系统 | 订单、AQL、抽样、主要/次要/严重缺陷、照片、结论 | 每批 | 出货放行或返工 | 供应链、仓库、客户 |
| 尺寸测量表 | Excel | 尺码、部位、标准值、公差、实测值、偏差、判定 | 每批 | 控制版型落差 | QC、生产、商品 |
| 缺陷照片库 | 图片库/知识库 | 缺陷名称、位置、严重等级、原因、处理方法、示例图 | 持续维护 | 统一判断标准 | QC、供应商 |
| 返工整改通知 | 邮件/工单 | 问题描述、责任方、整改动作、完成期限、复检要求 | 触发式 | 推动纠正 | 工厂、生产协调 |
| 复检报告 | PDF/系统 | 原问题、整改结果、复检抽样、是否放行、遗留风险 | 触发式 | 闭环质量异常 | 供应链、仓储 |
| 质量周报 | PPT/Excel | 不合格率、主要缺陷、供应商排行、重点风险、整改状态 | 每周 | 质量管理会议 | 供应链、采购 |
| 测试送检申请 | 系统/表单 | 标准、项目、样品、批次、实验室、预计出报告日 | 触发/每季 | 化学/物性合规 | 采购、法务 |
| 质量扣款建议 | Excel/邮件 | 供应商、订单、问题、返工费用、扣款依据、金额 | 触发式 | 商务索赔 | 采购、财务 |
| 质量标准/SOP | Word/PDF | 验货流程、AQL 水平、缺陷等级、公差、包装标准 | 季度/年度 | 培训和执行 | QC、供应商 |
| 客诉质量分析 | Excel/PPT | 退货原因、图片、批次、供应商、责任归因、改进建议 | 每月/触发 | 降低复发 | 客服、商品、供应链 |

## 6. 关键决策点

| 决策内容 | 决策依据 | 决策错误的代价 |
|---|---|---|
| 该批货是否放行 | AQL 结果、严重缺陷、客户标准、销售节点、返工可行性 | 误放会引发退货和品牌损害；误拒会造成延期和成本。 |
| 缺陷属于主要还是次要 | 功能影响、安全风险、外观可见性、客户标准 | 评级错误会导致过度返工或漏放风险。 |
| 是否要求全检 | 抽检结果、订单重要性、历史问题、缺陷分布 | 全检成本高，不全检可能漏掉批量问题。 |
| 测试不合格是否可重测或换料 | 标准要求、检测误差、用途、法规风险 | 合规问题放行可能触法或召回。 |
| 质量责任归谁 | 封样、采购标准、工艺单、生产记录、照片证据 | 归因错误会影响索赔和供应商关系。 |

## 7. 专业知识与术语

### 行业黑话 / 缩略语
AQL：可接受质量限；AC/RE：接收/拒收数；Major Defect：主要缺陷；Minor Defect：次要缺陷；Critical Defect：严重缺陷；公差；色差；缸差；缩水率；色牢度；克重；门幅；针距；跳针；漏针；爆线；污渍；破洞；扭斜；起球；洗水唛；主唛；吊牌；封样；首件；中查；尾查；全检；抽检；让步接收；复检；CAP：纠正预防措施。

### 常用 KPI 名词及计算口径
| KPI | 常用口径 |
|---|---|
| 一次验货通过率 | 首次验货合格批次/验货批次 |
| 返工率 | 返工件数/检验件数 |
| 客诉率 | 质量客诉件数/销售件数或订单数 |
| 来料合格率 | 合格来料批次/来料检验批次 |
| 整改关闭率 | 按期关闭异常数/异常总数 |

### 日常使用的软件 / 系统 / 模板
QMS、ERP 质检模块、PLM 标准样资料、图片库、第三方检测平台、Excel 抽样表、移动验货 App。

### 必懂的行业基础知识
必须掌握服装常见缺陷、尺寸公差、AQL 抽样、GB 18401 等基础安全要求、洗护标签、包装要求和供应商整改流程。

## 8. 痛点 / 低效环节
- 验货照片、纸质记录和系统报告割裂，后续追溯困难。
- 不同 QC 对缺陷等级理解不一致。
- 销售节点紧时放行压力大，质量判断容易被商业压力干扰。
- 客诉反馈到生产和采购太慢，重复问题难根治。
- 标准样、工艺单版本混乱导致验货依据不清。

## 9. 典型用户问句
1. 这份验货报告里主要缺陷超标了吗？按 AQL 2.5 判断。
2. 帮我把这批羽绒服的客诉图片归类成质量原因。
3. 这批裤子腰围偏大 1.2cm，能让步放行吗？
4. 帮我写一份给工厂的整改通知。
5. GB 18401 对贴身童装有哪些基础安全要求？
6. 这批洗水唛成分标错，风险等级怎么判？
7. 把本月供应商质量排名做成周会页。
8. 这款退货原因里“尺码偏小”是真质量问题还是版型反馈？
9. 帮我整理针织衫常见验货缺陷清单。
10. 复检通过但仍有轻微色差，出货备注怎么写？
