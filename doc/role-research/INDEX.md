# INDEX：服装公司非实物产出岗位职能与产物画像总览

> 范围：中等规模中国/全球服装公司，约 100-2000 人、年营收 1-50 亿；排除设计、版师、样衣、打版、车缝等直接实物产出岗位。本总览和各角色文件用于多 agent AI 协作系统的角色 prompt 和 output contract 原料。公开来源包括 BOSS/猎聘招聘 JD、服装 ERP/PLM/OMS/WMS 厂商说明、平台指标说明、国家纺织标准与行业报告，例如 [BOSS 商品企划 JD](https://www.zhipin.com/baike/b140312/fee14d22cce45d950nZ82Nm8F1c~.html)、[百胜服饰 ERP](https://www.baison.com.cn/?a=index&aid=2664&c=View&m=home)、[Centric Fashion PLM](https://www.centricsoftware.com/fashion-apparel)、[Oracle Fashion Supply Chain](https://www.oracle.com/retail/fashion/fashion-supply-chain/)、[McKinsey State of Fashion 2025](https://www.mckinsey.com/industries/retail/our-insights/state-of-fashion-2025)。

## 合并 / 拆分说明

1. **采购合并**：面料、辅料、成品采购在大公司会拆岗，但共同产物高度相似：供应商库、RFQ、比价、PO、交期、合同、对账。因此合并为 `role-03-procurement-sourcing.md`，在 agent prompt 中可再按品类加专业知识库。
2. **商品企划与商品运营拆分**：前者偏“开发前的货盘策略”，后者偏“上市后的进销存动作”。公开 JD 中二者常混写，但 AI agent 的输出契约差异很大，所以拆开。
3. **品牌营销、PR、内容社媒拆分**：品牌营销做策略和 campaign，PR 做第三方声量和风险，内容社媒做日常内容产出。三者产物、审批和风险完全不同。
4. **数据 BI 与 IT 数据中台拆分**：BI 负责指标、看板和分析；IT/数据中台负责系统、接口、权限、主数据和上线运维。
5. **HR 与培训/OD 合并**：中型服装公司常将招聘、绩效、零售培训和组织发展放在 HR 部门内，培训内容又高度依赖商品和门店，因此合并。

## D. 关键 KPI 词典

| KPI | 统一定义/口径 | 主要使用岗位 | 备注/来源 |
|---|---|---|---|
| 销售额 | 按业务口径可分吊牌销售、成交销售、退款前 GMV、退款后净销售，必须在报表标注。 | 商品运营、渠道、电商、财务、BI | 平台 GMV 和财务收入常不同，需口径字典。 |
| GMV | 平台成交金额，通常为成交订单金额，是否扣退款按平台/公司口径注明。 | 电商、营销、财务、BI | 参考 [抖音电商指标定义](https://school.jinritemai.com/doudian/web/article/104021)。 |
| 售罄率 | 累计销售件数 / 累计到货件数。 | 商品企划、商品运营、渠道、门店 | 服装季节性强，需按上市天数分组。 |
| 动销率 | 有销售 SKU 数 / 可售 SKU 数。 | 商品企划、商品运营、门店 | 可按款、款色、款色码口径分别计算。 |
| 库销比 | 期末库存数量或金额 / 近期销售数量或金额。 | 商品运营、渠道、仓储、财务 | 近期销售周期常用 7/14/30 天，需写明。 |
| 库存周转率 | 销售成本 / 平均库存成本；零售数学常用该公式。 | 商品、财务、仓储、管理层 | 参考 [Shopify Retail Math](https://www.shopify.com/blog/retail-math)。 |
| 周转天数 | 平均库存 / 日均销售成本或日均销量。 | 商品、财务 | 季节款应结合生命周期判断。 |
| 毛利率 | 毛利 / 销售额；毛利 = 销售额 - 销售成本。 | 财务、商品、电商、渠道 | 活动需扣平台费、佣金、物流、退货等经营成本。 |
| 折扣率 | 成交价 / 吊牌价。 | 商品、渠道、电商、门店 | 也可用 1-折扣率 表示折让。 |
| 客单价 | 销售额 / 成交单数或支付买家数。 | 门店、电商、BI | 平台口径以支付买家或订单为准。 |
| 转化率 CVR | 成交人数或订单 / 访客或观看人数。 | 电商、门店、内容 | 线上可参照 [Google Analytics 维度指标](https://support.google.com/analytics/table/13948007?hl=en) 和平台后台。 |
| 点击率 CTR | 点击数 / 曝光数。 | 电商、内容、投放 | 常用于封面、广告、商品卡和搜索。 |
| 退货率 | 退货订单/支付订单，或退货金额/支付金额。 | 客服、电商、商品、财务 | 服装需按尺码、面料、渠道拆分。 |
| 准交率 | 按计划交付批次或订单 / 应交付批次或订单。 | 采购、供应链、QC | 可分物料准交和大货准交。 |
| 一次验货通过率 | 首次验货合格批次 / 首次验货批次。 | QC、供应链、采购 | AQL 抽样和缺陷等级参考 [服装 AQL 抽样说明](https://www.gis-inspection.com/zh/post/garment-Inspection-aql-sampling-inspection)。 |
| 库存准确率 | 账实一致 SKU 或数量 / 盘点 SKU 或数量。 | 仓储、财务、IT | WMS 流程参考 [Kladana WMS Process Flow](https://www.kladana.com/blog/wms/wms-process-flow/)。 |
| 回款率 | 已回款金额 / 应回款金额。 | 渠道、财务 | 需结合账期和逾期天数。 |
| 人效 | 销售额、订单数或产出 / 人数或人时。 | 门店、客服、仓储、HR | 不同岗位需独立口径。 |
| 满意度 | 满意评价 / 有效评价，或平台客服满意度。 | 客服、门店、HR | 与响应时效、解决率联动。 |

## 推荐的 agent delegate 路径提示

- **库存/售罄/补调退问题**：商品运营 → 仓储物流 → 电商/门店/渠道；需要成本时委托财务，需要预测时委托 BI。
- **新品上市问题**：商品企划 → 品牌营销 → 内容/PR/电商/门店；涉及交期时委托生产协调，涉及物料时委托采购。
- **质量/退货问题**：客服 VOC → QC → 生产协调/采购；涉及消费者争议时委托法务。
- **活动/大促问题**：电商运营或渠道运营 → 商品运营 → 仓储物流 → 财务 → 内容/客服。
- **系统/数据问题**：业务岗位 → 数据 BI 判定指标口径 → IT 数据中台处理接口、权限和主数据。
