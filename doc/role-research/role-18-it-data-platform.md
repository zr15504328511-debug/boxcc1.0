## 1. 岗位标识
- **中文名 / 英文名 / 常见别名**：IT / 数据中台 / IT / Data Platform / Digital Systems / 信息化、ERP/PLM/WMS/OMS 产品经理、系统管理员、数据中台、业务中台。
- **所属部门 / 汇报关系**：IT 部 / 数字化中心 / 数据中台；汇报 CIO、信息化总监、数字化负责人或 CFO。
- **在组织中的层级位置**：数字化基础设施岗位，负责服装企业 ERP、PLM、OMS、WMS、POS、CRM、BI 等系统建设和运维。
- **与设计 / 版师 / 样衣 / 打版岗位的边界**：不做业务决策、不设计服装；IT 提供流程、数据和工具能力，业务部门定义规则和最终动作。

## 2. 核心职能
1. 保障 ERP、PLM、OMS、WMS、POS、CRM、BI 等核心系统稳定运行和数据流转。
2. 把业务流程抽象成系统需求、权限、字段、接口和报表，减少人工重复。
3. 维护商品、客户、供应商、库存等主数据标准，保证全链路可追踪。
4. 推动系统上线、培训、变更管理和问题闭环，降低业务中断。
5. 与 BI、业务和供应商协作，建设数据中台和接口能力。

## 3. 典型工作节奏
- **日常**：处理系统故障、权限、数据同步、报表错误、账号和业务咨询。；跟进需求评审、开发、测试、上线和供应商响应。
- **周期性**：每周需求池和问题池评审；每月系统稳定性、数据质量和项目进度复盘。；每季做主数据治理、权限审计、系统优化和培训。
- **触发式**：大促、订货会、新店开业、系统升级、接口异常、数据泄漏风险、业务流程变化时触发专项保障。

## 4. 协作关系
- **上游**：业务部门提出流程和报表需求；软件供应商提供系统能力；BI 提出数据要求；法务/财务给合规和内控要求。
- **下游**：全公司依赖系统权限、流程、数据、看板和问题支持。
- **横向协作最频繁的 3 个岗位**：数据/BI、商品运营、仓储物流。

## 6. 关键决策点

| 决策内容 | 决策依据 | 决策错误的代价 |
|---|---|---|
| 需求优先级怎么排 | 业务价值、紧急度、风险、开发成本、依赖关系 | 排错会影响关键业务或拖慢数字化。 |
| 流程标准化还是定制化 | 行业最佳实践、业务差异、维护成本、系统能力 | 过度定制难维护，过度标准无法落地。 |
| 数据主键和编码规则怎么定 | 款色码结构、系统兼容、历史数据、扩展性 | 编码混乱会影响全链路数据。 |
| 上线是否延期 | 测试结果、缺陷等级、业务窗口、回滚能力 | 强行上线可能中断业务，延期影响项目收益。 |
| 权限是否开放 | 岗位职责、数据敏感度、内控、效率 | 权限过大有泄露和舞弊风险。 |

## 8. 痛点 / 低效环节
- 老系统和新平台接口复杂，库存和订单同步易出错。
- 业务需求口头化，缺少可验收规则。
- 主数据治理枯燥但影响全链路，责任常不清。
- 系统上线时培训不到位，用户绕回 Excel。
- 大促高峰下系统容量和接口稳定性压力大。
