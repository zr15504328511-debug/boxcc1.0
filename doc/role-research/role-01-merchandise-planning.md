# 商品企划（Merchandise Planning / Product Planning）

> 调研说明：本画像用于为多 agent AI 协作系统提供角色 prompt 原料。内容基于公开招聘 JD、服装/零售系统厂商说明、行业资料交叉归纳；不同公司会因品牌定位、渠道结构和规模不同而调整权责。公开信息有限处已按中等规模服装品牌的常见组织做行业类比。

## 1. 岗位标识
- **中文名 / 英文名 / 常见别名**：商品企划 / Merchandise Planning / Product Planning / 商品规划、产品企划、品类企划、买手企划、OTB 计划。
- **所属部门 / 汇报关系**：商品中心 / 产品中心 / 品牌事业部；通常汇报商品总监、品牌总经理或品类 BU 负责人；向设计开发、供应链、渠道运营提供输入。
- **在组织中的层级位置**：中台策略岗位，介于品牌战略、设计开发和销售经营之间；在 100-2000 人公司常为 1-6 人小组。
- **与设计 / 版师 / 样衣 / 打版岗位的边界**：不画款式图、不做版型结构、不打版、不制作样衣；只定义品类、价格带、波段、数量和商业角色，并把需求转成企划文档。

## 2. 核心职能
1. 把品牌战略和销售目标翻译成“本季该卖什么、卖多少、何时上市、什么价格”的商品结构。
2. 用历史销售、竞品、趋势和渠道反馈控制新品宽度、深度、波段和价格带，避免盲目开发。
3. 在设计开发前明确商品商业角色，如形象款、引流款、利润款、走量款、补充款。
4. 在订货、上市、补单、清仓之前给出经营假设和风险边界。
5. 维护商品生命周期语言，让设计、采购、生产、销售对同一盘货的目标一致。

## 3. 典型工作节奏
- **日常**：查看昨日销售、库存、畅滞销、竞品和渠道反馈，更新企划假设。；回答设计、运营、供应链关于品类数、上市波段、价格带和主推款的问题。
- **周期性**：每周复盘新品表现、在途与库存风险；每月调整品类和价格策略。；每季输出商品企划案、开发需求、订货会材料；每年参与年度商品预算和品类战略。
- **触发式**：销售突变、竞品爆款、平台大促、面料断供或天气异常时，触发补单、砍款、延期上市或促销建议。

## 4. 协作关系
- **上游**：经营目标来自老板/品牌总监/财务预算；销售与库存来自商品运营/BI；趋势和竞品来自市场、内容、渠道。
- **下游**：设计开发、采购、供应链、销售渠道、电商、培训都依赖企划确定货盘和节奏。
- **横向协作最频繁的 3 个岗位**：商品运营/分析、供应链生产协调、品牌营销。

## 6. 关键决策点

| 决策内容 | 决策依据 | 决策错误的代价 |
|---|---|---|
| 本季每个品类开发多少款 | 历史销售贡献、目标增长、渠道能力、SKU 管理成本 | 宽度过大导致研发和库存压力；过窄导致选择不足和缺货。 |
| 价格带是否上移或下探 | 竞品价格、目标客群、成本倍率、渠道折扣空间 | 价格错位会影响转化、毛利和品牌定位。 |
| 主推款和形象款怎么分配 | 品牌调性、广告资源、预估销量、供应难度 | 主推错会浪费投放和店铺陈列资源。 |
| 某款是否补单或砍单 | 动销、售罄、库销比、面辅料交期、销售窗口 | 补错造成库存；砍错造成断码断货和销售损失。 |
| 上市波段是否提前/后延 | 天气、平台活动、工厂交期、竞品上新节奏 | 错过销售窗口或货品季节不匹配。 |

## 7. 专业知识与术语

### 行业黑话 / 缩略语
OTB：可采买预算；宽度：SKU 款数；深度：单款订量；波段：分批上市节奏；价格带：吊牌价区间；售罄率：累计销售/到货；动销率：有销售 SKU/可售 SKU；库销比：库存/近期销售；生命周期：新品到清仓全过程；主推款：资源集中款；形象款：体现品牌调性款；利润款：高毛利款；引流款：低价拉新款；跑量款：承担销量款；SKU：款色码最小销售单元；SPU：款式维度；倍率：吊牌价/成本；畅销/滞销：相对计划表现；订货会：渠道集中订货会议；企划命中率：实际表现与企划目标吻合度。

### 常用 KPI 名词及计算口径
| KPI | 常用口径 |
|---|---|
| 企划命中率 | 实际销售/企划销售，或实际畅销款占预测畅销款比例 |
| 售罄率 | 累计销售件数/累计到货件数 |
| 毛利率 | 毛利/销售额 |
| 新品销售占比 | 新品销售额/总销售额 |
| 库存周转 | 销售成本/平均库存成本，或平均库存/日均销售 |

### 日常使用的软件 / 系统 / 模板
Excel、PPT、PLM、ERP、BI 看板、竞品监测工具、生意参谋/抖店罗盘/渠道销售报表、飞书/钉钉评审表。

### 必懂的行业基础知识
理解品类角色、季节性、款色码结构、价格倍率、面辅料和工艺对成本/交期的影响，但不替代设计和版师做实物开发。

## 8. 痛点 / 低效环节
- 销售、库存、设计开发和供应链数据口径不一，企划案常靠手工拼表。
- 趋势信息多但难转成可执行款数、价格带和波段。
- 企划与上市结果之间缺乏闭环，复盘常停留在“爆了/没爆”。
- 老板和渠道临时改方向，导致开发资源反复摇摆。
- 历史数据无法解释新渠道和新内容场景，判断依赖个人经验。

## 9. 典型用户问句
1. 帮我把 2025 秋冬羽绒品类按价格带和渠道销售拉一版复盘。
2. 这季女装开发 180 款是不是太宽？按去年售罄帮我压到 140 款。
3. 这款针织衫定位利润款还是跑量款更合适？
4. 根据直营和天猫数据，帮我给双11前的上市波段重排一下。
5. 竞品最近都在推麂皮夹克，我们要不要临时加一个小胶囊？
6. 把这份设计提案转成商品企划会能看的品类结构页。
7. 这款 SKU 库销比 8 了，要不要进入清仓？
8. 帮我评估秋款下单 5000 件的库存风险。
9. 莱赛尔混纺和纯棉同价位，哪个更适合做引流基础款？
10. 帮我写一段给销售会讲的本季货盘逻辑。
